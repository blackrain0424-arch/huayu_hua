## 📸 图片要求

### 文件命名规则
- 图片文件名必须与花朵名称完全一致
- 使用中文命名，例如：`梅花.jpg`、`牡丹.jpg`
- 支持的格式：`.jpg`、`.png`、`.jpeg`

### 图片规格建议
- **推荐尺寸**：至少 200x200 像素
- **最佳尺寸**：400x400 像素（方图）
- **文件大小**：建议小于 500KB
- **图片质量**：清晰、光线充足、背景简洁

### 图像要求
- **主体突出**：花朵应占据图片主要部分
- **构图美观**：居中或黄金分割构图
- **色彩还原**：真实还原花朵本色
- **避免问题**：
  - 模糊不清
  - 过度曝光
  - 背景杂乱
  - 有水印或文字

## 🔧 如何添加图片

### 方法1：直接放置
1. 下载花朵图片（推荐从以下网站）：
   - Unsplash: https://unsplash.com/s/photos/flowers
   - Pexels: https://www.pexels.com/search/flowers/
   - Pixabay: https://pixabay.com/images/search/flower/

2. 重命名为对应的中文名称，例如：
   ```
   downloaded_photo_123.jpg → 梅花.jpg
   peony_flower.jpg → 牡丹.jpg
   ```

3. 将文件放到 `assets/flowers/` 目录下

## ✨ 工作原理

### 优先级逻辑
1. **优先显示图片**：如果 `imagePath` 对应的图片存在，显示真实图片
2. **降级到Emoji**：如果图片不存在或加载失败，自动显示对应的emoji图标
3. **无需修改代码**：只需添加图片文件，无需修改任何代码

### 自动适配
- **小图标**：卡片中的小图标（56x56）
- **大图标**：详情页中的大图标（72x72）
- **圆角处理**：自动添加圆角效果
- **错误处理**：图片加载失败自动回退到emoji

## 💡 提示

1. **部分添加**：不需要一次性添加所有图片，可以逐步添加
2. **测试效果**：添加后重新运行APP即可看到效果
3. **热重载**：使用热重载（`r`）快速预览图片效果
4. **图片质量**：建议使用高质量图片以获得最佳显示效果

## 🔄 更新步骤

```bash
# 1. 添加图片到 assets/flowers/ 目录

# 2. 如果新增了花种，需要更新以下文件：
#    - lib/data/daily_flower_data.dart（添加新的DailyFlower对象）
#    - lib/models/daily_flower.dart（如需修改模型）

# 3. 运行APP
flutter run

# 或热重载
r
```

## 📝 注意事项

1. **文件名必须完全匹配**（区分大小写）
2. 图片格式必须是 `.jpg`、`.png` 或 `.jpeg`
3. 图片路径已在代码中配置为 `assets/flowers/文件名.jpg`
4. 如需更改路径格式，请同步修改 `lib/data/daily_flower_data.dart`
